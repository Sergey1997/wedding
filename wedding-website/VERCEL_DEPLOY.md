# Деплой на Vercel

## Быстрый деплой

1. **Подключите репозиторий к Vercel:**
   - Зайдите на [vercel.com](https://vercel.com)
   - Нажмите "Add New Project"
   - Импортируйте ваш GitHub репозиторий

2. **Настройки проекта:**
   - **Framework Preset:** Vite
   - **Build Command:** `pnpm build` (или оставьте пустым, используется из vercel.json)
   - **Output Directory:** `dist`
   - **Install Command:** `pnpm install` (или оставьте пустым)

3. **Добавьте переменные окружения:**
   - В настройках проекта перейдите в **Environment Variables**
   - Добавьте:
     - `VITE_SUPABASE_URL` = ваш Supabase URL
     - `VITE_SUPABASE_ANON_KEY` = ваш Supabase anon key
   - Выберите окружения: **Production**, **Preview**, **Development**

4. **Деплой:**
   - Нажмите "Deploy"
   - Vercel автоматически определит pnpm из `package.json` и `vercel.json`

## Проверка деплоя

После успешного деплоя:
- ✅ Сайт должен открываться
- ✅ Форма RSVP должна работать (если настроен Supabase)
- ✅ Все изображения должны загружаться

## Решение проблем

### Ошибка "vite: command not found"

✅ **Исправлено:** Добавлен `vercel.json` с указанием pnpm и команды сборки

### Ошибка сборки TypeScript

Если есть ошибки типов, они будут показаны в логах Vercel. Исправьте их локально перед деплоем.

### Переменные окружения не работают

Убедитесь что:
- Переменные добавлены для всех окружений (Production, Preview, Development)
- Имена переменных начинаются с `VITE_` (это обязательно для Vite)
- После добавления переменных пересоберите проект

## Автоматический деплой

После первого деплоя, Vercel будет автоматически деплоить при каждом push в:
- `master`/`main` → Production
- Другие ветки → Preview

## Кастомный домен

1. В настройках проекта → **Domains**
2. Добавьте ваш домен
3. Следуйте инструкциям по настройке DNS
